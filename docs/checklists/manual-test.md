# 手動テストチェックリスト

本番デプロイ前に確認すべき手動テスト項目。
開発サーバー (`npm run web`) を起動した状態で実施する。

---

## 1. ログイン画面（未認証状態）

### 1.1 表示確認
- [ ] 「生駒祭 2026」「情宣AI判定システム」が表示される
- [ ] 「ログイン」「新規登録」タブが表示される
- [ ] メールアドレス入力欄（placeholder: `example@kindai.ac.jp`）が表示される
- [ ] パスワード入力欄（placeholder: `6文字以上`）が表示される
- [ ] フッター「大学祭実行委員会 情宣局」が表示される

### 1.2 タブ切替
- [ ] 初期状態で「ログイン」タブがアクティブ（青背景）
- [ ] 「新規登録」タブをクリックすると切り替わる
- [ ] 送信ボタンが「ログイン」⇔「登録」に変わる
- [ ] タブ切替でエラーメッセージがクリアされる

### 1.3 バリデーション
- [ ] 空欄で送信 → 「メールアドレスとパスワードを入力してください。」
- [ ] `test@gmail.com` で送信 → 「@kindai.ac.jp のメールアドレスを使用してください。」
- [ ] 新規登録モードでパスワード5文字以下 → 「パスワードは6文字以上で入力してください。」

### 1.4 レスポンシブ
- [ ] デスクトップ（≥1024px）で正常にレイアウトされる
- [ ] モバイル（<768px）でフォーム幅が適切に狭まる

## 2. 新規登録フロー

### 2.1 正常系
- [ ] `@kindai.ac.jp` のメールアドレスで新規登録できる
- [ ] メール確認が有効な場合：成功メッセージが表示される
- [ ] メール確認が無効な場合：自動ログインしてメイン画面に遷移する

### 2.2 異常系

## 3. ログインフロー

### 3.1 正常系
- [ ] 正しいメール/パスワードでログインできる
- [ ] ログイン後にサンドボックス画面に遷移する
- [ ] ローディングスピナーが送信中に表示される

### 3.2 異常系
- [ ] 間違ったパスワード → 「メールアドレスまたはパスワードが正しくありません。」

### 3.3 プロフィール自動作成
- [ ] 初回ログイン時に `user_profiles` にレコードが作成される
- [ ] 初回ログイン時に `josenai_profiles` にレコードが作成される
- [ ] 表示名がメールアドレスの `@` 前部分になっている

## 4. 管理者認証

### 4.1 モーダル表示
- [ ] ドロワーメニューから管理者認証モーダルを開ける
- [ ] 3つの権限種別（広報部 / 企画管理部 / 管理者）が選択できる
- [ ] 「スキップ」で一般ユーザーのまま閉じられる

### 4.2 認証成功
- [ ] 広報部パスワードで認証 → ダッシュボード・ルール・設定メニューが表示される
- [ ] 企画管理部パスワードで認証 → 同上
- [ ] 管理者パスワードで認証 → 上記 + マスタ管理メニューが表示される

### 4.3 認証失敗
- [ ] 間違ったパスワード → 「パスワードが正しくありません。」
- [ ] Edge Function エラー時 → 「認証処理中にエラーが発生しました。」

### 4.4 セッション動作
- [ ] ページリロード後に権限がリセットされる（一般ユーザーに戻る）
- [ ] ログアウト後に権限がリセットされる

## 14. Edge Functions 接続確認

| 関数 | 確認方法 |
|------|----------|
| `submit` | 正式提出を実行して成功を確認 |
| `sandbox` | サンドボックスでAI判定を実行 |
| `delete-submission` | 提出を削除して Drive + DB 両方から消えることを確認 |
| `verify-admin-password` | 管理者認証モーダルで認証を実行 |
| `update-password` | 設定画面でパスワード変更を実行 |
| `review` | ダッシュボードで承認/却下を実行 |
| `import-organizations` | マスタ管理で団体 CSV をインポート |
| `import-projects` | マスタ管理で企画 CSV をインポート |
